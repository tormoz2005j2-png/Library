# Моя библиотека

Личная библиотека с облачным хранением в Cloudflare D1. Статическая часть может размещаться на GitHub Pages, а доступ к базе идёт только через защищённый Cloudflare Worker.

API уже развёрнут: `https://library-api.tormoz2005j2.workers.dev`. База создана в регионе EEUR, схема применена, секрет настроен. Личный ключ сохранён в Связке ключей macOS под именем `Biblioteka Cloud API` и скопирован в буфер обмена. Повторно скопировать его можно так:

```bash
security find-generic-password -a tormoz2005j2-png -s "Biblioteka Cloud API" -w | pbcopy
```

## Как устроено хранение

- `library_items` — отдельная строка для каждого тайтла с проверками типов, статусов, оценок и денежных значений.
- `library_settings` — выбранная валюта и признак первичной миграции.
- `LIBRARY_TOKEN` — личный ключ API. Он хранится как секрет Worker и не попадает в репозиторий.
- В браузере ключ находится только в `sessionStorage` и удаляется после закрытия вкладки.
- Старый `localStorage` читается единственный раз при пустой БД и переносится в D1. Новые изменения туда не записываются.
- `localStorage` привязан к адресу страницы. Если старая версия открывалась с другого адреса или из файла, сделайте в ней «Экспорт», а затем «Импорт» в облачной версии.
- «Экспорт» остаётся независимой JSON-резервной копией, «Импорт» полностью заменяет содержимое базы.

## Первое развёртывание

Понадобятся Node.js 20+ и бесплатный аккаунт Cloudflare.

```bash
cd worker
npm install
npx wrangler login
npm run db:create
```

Команда создания D1 напечатает `database_id`. Вставьте его вместо `PASTE_DATABASE_ID_HERE` в `worker/wrangler.jsonc`.

Создайте таблицы:

```bash
npm run db:init:remote
```

Создайте случайный ключ и сохраните его в менеджере паролей:

```bash
openssl rand -base64 32
npm run secret
```

Во второй команде вставьте сгенерированный ключ. Не добавляйте его ни в один файл проекта.

Опубликуйте API:

```bash
npm run deploy
```

API этого проекта опубликован по адресу `https://library-api.tormoz2005j2.workers.dev` и уже указан в `api-config.js`:

```js
window.LIBRARY_CONFIG = Object.freeze({
  apiUrl: "https://library-api.tormoz2005j2.workers.dev"
});
```

В `worker/wrangler.jsonc` параметр `ALLOWED_ORIGINS` должен содержать origin сайта без пути. Для GitHub Pages этого репозитория это `https://tormoz2005j2-png.github.io`.

## Локальная проверка сайта

Страницу нужно открывать через HTTP, а не двойным кликом по файлу:

```bash
python3 -m http.server 8000
```

Затем откройте `http://localhost:8000`. При первом запросе браузер попросит личный ключ API.

## Безопасность и восстановление

- CORS ограничивает браузерные запросы известными адресами, а секретный ключ защищает сам API.
- Ключ сравнивается по SHA-256, чтобы не использовать прямое сравнение строк.
- Ответы API запрещено кешировать.
- Секрет можно немедленно заменить командой `npm run secret` и повторной публикацией Worker.
- Для восстановления используйте последний JSON-экспорт через кнопку «Импорт».
